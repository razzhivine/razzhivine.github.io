Проблематика и её решение
До 15% времени разработчиков тратиться на организацию размещения ресурсов
Для бизнеса
Пример
Оперативное развертывание ВМ
  • Много времени на уточнение сетевых связанностей и др. нюансы
  • Развёртывание происходило в ручную
Решение
Механизм самообслуживания
Автоматическое развертывание сред для тестирования, разработки и продуктивного контура
20% всех ресурсов простаивают из-за невозможности их переиспользования в моменте времени
Для бизнеса
Пример
Необходимость гибкого перераспределения ресурсов между системами
Решение
Учет и перераспределение с помощью единой консоли и управление динамической инфраструктуры
что позволяет экономить до 15% и более затрат.
Лавинообразные нагрузки
Для бизнеса
Пример
Увеличение инфраструктуры на 30%
Предоставление ресурсов on demand, infrastructure as a code, контейнерные кластеры
Решение
Получение льгот, введение особых регламентов, электоральная автивность и т.д
Слишком много сил тратится на обеспечение качества и оперативности релизов
Для операционной деятельности
Пример
Несколько сред
  • Разрывы на стыках процессах
  • Человеческий фактор
Решение
Полная поддержка лучших практик и инструментариев CI\CD
Требуется 2-3 итерации для подбора и размещения ресурсов на старте проекта (мин. 1-2 мес.)
Для операционной деятельности
Пример
Выделение второго контура и вынос БД на baremetal
Решение
Гибкий предустановленный пул ресурсов и мгновенное развертывание необходимых сред
Заявки даже на тестовые ресурсы сложны и понятны только системным администраторам
Для операционной деятельности
Пример
Перегруз разработчика заполнением заявок
Вынужден знать, как устроены его тестовые среды, с точки зрения инфраструктуры и сетевой связанности
Решение
Работа с инфраструктурой, как с кодом и через REST API
  • Работа с инфраструктурой происходит на языке и в инструментах понятных и удобных для разработчиков
  • Нет необходимости погружения деталей реализации
  • Большой репозитарий шаблонов – сделай раз используй многократно
